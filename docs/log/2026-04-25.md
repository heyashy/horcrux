- [CHORE] Phase 2 — data modelling (cleansing + chapter detection).
  Files added:
  - `horcrux/cleansing.py` — page-level normalisation: strip standalone
    page-number lines, normalise `--` → `—`, collapse runaway newlines,
    trim outer whitespace. Idempotent. List-based API preserves page
    numbers (no dropping at this layer; chapter detection downstream
    decides what to skip).
  - `tests/unit/test_cleansing.py` — 11 tests covering each rule + edge
    cases (numbers in prose preserved, runaway newlines collapsed,
    em-dash normalisation idempotent).
  - `horcrux/models.py` extended with `Chapter`, `ChapterChunk`, and
    `make_chunk_id` helper. `ChapterChunk.id` is a deterministic UUID5
    derived from `(book_num, chapter_num, chunk_index, chunk_type)` so
    Qdrant upsert is naturally idempotent across re-ingest runs (ties
    back to the activity-idempotency discussion from the Temporal session).
  - `horcrux/chapters.py` — chapter and book detection. Initial
    implementation used regex-on-OCR-text scanning for `CHAPTER N`
    markers + chapter-reset heuristic for book boundaries. Pivoted to
    PDF embedded-outline extraction via `pymupdf.Document.get_toc()`
    after realising the PDF's bookmarks are authoritative structural
    metadata. Regex approach is the documented fallback for TOC-less
    corpora. See ADR-0005.
  - `tests/unit/test_chapters.py` — 13 tests on the pure
    `chapters_from_toc` function plus `_parse_chapter_title`. No PDF
    fixtures needed; synthetic TOC entries.
  - `scripts/inspect_chapters.py` — runs cleansing → chapter extraction
    on the cached OCR output, prints a Rich table per book with
    detected chapters, expected pages from the TOC ground truth, and
    drift indicator. Verified **15/15 chapters of book 1 match the TOC
    exactly** (zero drift). Also detects 16 chapters of book 2 (book 2
    chapter 16 is partial — OCR ran to page 520, chapter 16 ends at 535).

- [CHORE] Character discovery iteration journey on book 1's data
  (captured in detail in ADR-0006):
  - V1 (regretted): subset matching for any pair → "Weasley" mega-cluster
    pulled in every Weasley + Malfoy pulled Lucius+Draco.
  - V2: dropped single-to-multi subset → cleaner but lost some recall.
  - V3: added fastcoref Tier 1b → catastrophic over-attribution (Dumbledore
    cluster contained "He-Who-Must-Not-Be-Named", "Mr. Dursley", "Father";
    Harry cluster contained "Peeves"; Flitwick cluster contained "Fluffy").
  - V4: rejected multi-PERSON coref clusters; matched NER PERSON by text
    not just span. Cleaner but still let through "Fluffy" → Flitwick.
  - V5: added share-significant-token gate (alias and target must share
    at least one non-generic token). Blocks semantic mismatches; allows
    orthographic title-forms. Plus stricter alias-candidate filter
    (≤4 words, no commas/quotes/conjunctions).
  - V6: structure-preserving page join (above).
  - Final Tier 1 quality: 117 clusters from book 1 + book 2 partial OCR,
    clean orthographic + title-form merges (Albus Dumbledore →
    Dumbledore, Mr. Filch → Filch, Aunt Petunia → Petunia, Lord Voldemort
    → Voldemort, etc.). Single-word ↔ multi-word merges (Harry ↔ Harry
    Potter) deferred to Tier 2; semantic aliases (Voldemort ↔ Tom Riddle
    ↔ He Who Must Not Be Named) also deferred.
